What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web pages. It represents the structure of an HTML or XML document as a tree of objects, where each element (like a heading, paragraph, or image) is a node. This allows developers to access, modify, or update the content, structure, and style of a webpage using languages like JavaScript.

How do you select an element from the DOM?

To select an element from the DOM, we can use methods like- "getElementById('id')", "getElementsByClassName('class')", "getElementsByTagName('tag')" and many more

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is added to a parent element instead of multiple child elements. It takes advantage of event bubbling, where events triggered on child elements "bubble up" to their parent elements.

Why it's useful:

  • Efficiency: Instead of adding an event listener to each child element, you can manage events from the parent, reducing the number of event listeners.

  • Dynamic elements: It works well with elements added dynamically to the page, as the parent can handle events for new child elements without needing additional listeners.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes in the DOM, you can get or set attributes like id, src, or href using methods such as `getAttribute` or `setAttribute`. You can also remove attributes with `removeAttribute`.

For styles, you can directly modify the `style` property of an element to change things like its color, background, or size. Additionally, classes can be managed with `classList`, where you can add, remove, or toggle CSS classes to control the element's appearance.

These techniques allow you to dynamically change the design and behavior of elements on your website.
